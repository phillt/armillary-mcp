armillary-mcp

Stop AI From Rewriting Code You Already Have

Armillary builds a structured index of your exported TypeScript symbols — signatures and JSDoc included — and exposes it through MCP.

Agents search your codebase first. Then they write code.

Get Started Read the Manual

Features

Symbol Extraction

Every exported function, class, type, interface, enum, and constant — so agents can see what already exists.

Full Signatures

Complete type signatures so agents know exactly how to call existing code without reading source files.

JSDoc Parsing

@param, @returns, descriptions, and custom tags — so agents understand what code does without reading source.

CLI Tools

build generates the index, watch keeps it current as your code changes.

MCP Server

docs.list, docs.get, and docs.search — agents query your codebase through standard MCP tools.

Deterministic Output

Sorted symbols, Zod-validated schema — reliable, consistent index across rebuilds.

Quick Start

  1. Install 
    npm install --save-dev armillary-mcp
    pnpm add -D armillary-mcp
  2. Build the documentation index 
    npx armillary-mcp build
  3. Add code-reuse instructions to your agent

    Tell your agent to check the documentation index before creating new code. See the manual for agent-specific prompt snippets.

  4. Connect your AI coding assistant

    For Claude Code, register the server with the CLI:

    claude mcp add --transport stdio armillary-mcp -- npx armillary-mcp-server

    For Cursor, Windsurf, and other MCP clients, add to your client's config file:

    {
  "mcpServers": {
    "armillary": {
      "command": "npx",
      "args": ["armillary-mcp-server"]
    }
  }
}
  5. Watch for changes 
    npx armillary-mcp watch

    Tip: Add the watch command to your dev script so the index always stays current while you work.

How It Works

  1. Reads your tsconfig.json

    The indexer locates your TypeScript configuration and resolves all source files in the project.

  2. Extracts symbols via ts-morph

    Each source file is parsed into an AST. Exported functions, classes, types, interfaces, enums, and constants are extracted along with their signatures, JSDoc comments, and parameter details.

  3. Outputs validated JSON

    Symbols are sorted deterministically and validated against a Zod schema. The result is written to .armillary-mcp-docs/index.json, ready for the MCP server to serve.

  4. Your AI agent queries the index

    When your AI coding assistant needs to write code, it uses docs.search to find relevant existing symbols, docs.list to browse what's available, and docs.get to understand how to use them — reusing your code instead of reinventing it.
